1
NASMの基本構文を手で書く
MOV / ADD / CMP / JMP / INT を理解する
週1〜2 ▶

アセンブリは「命令 宛先, 元」の形。まず画面に文字を出す最小コードを手書きしましょう。

; boot.asm — 画面に'A'を出して止まる (16bit)
[BITS 16]
[ORG 0x7C00]

    mov ah, 0x0E    ; BIOS テレタイプ出力
    mov al, 'A'     ; 出力する文字
    int 0x10        ; BIOS呼び出し

    jmp $           ; 無限ループで停止

times 510-($-$$) db 0   ; 510バイトまで0埋め
dw 0xAA55               ; ブートシグネチャ

nasm -f bin boot.asm -o boot.bin → qemu-system-x86_64 -drive format=raw,file=boot.bin

2
GDB + QEMUでレジスタを目で見る
stepi で1命令ずつ、register の変化を確認
週1 ▶

書いたコードをGDBで1命令ずつ実行して、AXの中身が変わる瞬間を見る。これが最高の学習方法です。

# ターミナル1: QEMUを一時停止状態で起動
qemu-system-x86_64 -drive format=raw,file=boot.bin -s -S

# ターミナル2: GDBで接続
gdb
(gdb) target remote :1234
(gdb) set architecture i8086
(gdb) break *0x7c00        # ブートローダの先頭でブレーク
(gdb) continue
(gdb) info registers       # 全レジスタ表示
(gdb) x/16xb 0x7c00       # メモリの中身を見る
(gdb) stepi                # 1命令実行
(gdb) stepi                # また1命令

MOVの前後でAXが変わるのを確認する→理解が一気に深まります。

3
スタックと関数呼び出しを理解する
PUSH / POP / CALL / RET、SP・BPの動き
週2 ▶

関数呼び出しはすべてスタックで実現されます。CALLは「戻り先アドレスをPUSHしてJMP」、RETは「スタックからアドレスをPOPしてJMP」です。

; 関数呼び出しの骨格
call my_func        ; SP-=2, [SP]=戻り先, JMP my_func

my_func:
    push bp         ; 古いBPを保存
    mov  bp, sp     ; 現在のSPをBPに記録
    ; … 処理 …
    pop  bp         ; BPを戻す
    ret             ; SP+=2, JMP [SP]

GDBで stepi しながら SP の値が変わるのを見ると完全に理解できます。

4
BIOS INT でI/Oを使う
INT 0x10（画面）INT 0x13（ディスク）INT 0x16（KB）
週2〜3 ▶

BIOSのサービスは「AHにサービス番号を入れてINT命令を呼ぶ」という共通作法です。

; 文字列を画面に表示
print_str:
    mov ah, 0x0E    ; サービス: テレタイプ出力
.loop:
    lodsb           ; AL = [DS:SI], SI++
    test al, al     ; AL == 0 ?
    jz .done
    int 0x10
    jmp .loop
.done:
    ret

; ディスク読み込み (INT 0x13 AH=0x02)
    mov ah, 0x02    ; サービス: セクタ読み込み
    mov al, 4       ; 読むセクタ数
    mov ch, 0       ; シリンダ0
    mov cl, 2       ; セクタ2から (1=ブートセクタ)
    mov dh, 0       ; ヘッド0
    mov bx, 0x8000  ; 読み込み先 ES:BX
    int 0x13
5
割り込みハンドラを自作する
IVTにアドレスを書き、タイマー・キーボードを捕まえる
週3〜4 ▶

IVT（0x0000〜0x003FF）は「割り込み番号 × 4」のアドレスにジャンプ先（セグメント:オフセット）が入っています。ここを書き換えると割り込みを乗っ取れます。

; IRQ0 (タイマー, INT 0x08) を乗っ取る
    cli                     ; 割り込み禁止
    mov ax, 0x0000
    mov es, ax
    mov word [es:0x08*4],   my_timer    ; オフセット
    mov word [es:0x08*4+2], cs          ; セグメント
    sti                     ; 割り込み許可

my_timer:
    pusha                   ; 全レジスタ保存
    ; … タイマー処理 …
    mov al, 0x20
    out 0x20, al            ; PICにEOI送信 (必須!)
    popa                    ; 全レジスタ復元
    iret                    ; 割り込みから復帰

out 命令でPICにEOI(End of Interrupt)を送らないと次の割り込みが来なくなります。ここは初心者がハマりやすいポイントです。

6
疑似マルチタスク（コンテキストスイッチ）
タイマー割り込みでレジスタを切り替える
週4〜6 ▶

「マルチタスク」の本質は「タイマー割り込みのたびにレジスタ一式（コンテキスト）を別のタスクのものに差し替えること」です。

; タスク構造体（各タスクのレジスタ保存領域）
; task_a_ctx: SP, BP, IP... を保存するメモリ領域
; task_b_ctx: 同上

my_timer:
    pusha
    ; 現在のタスクのSP等を task_X_ctx に保存
    ; 次のタスクの task_Y_ctx からSP等を復元
    popa
    mov al, 0x20
    out 0x20, al
    iret

これを理解した瞬間「OSって結局これか！」となります。16bitリアルモードにおける最大の達成感ポイントです。

7
（発展）プロテクトモードへ移行
GDT設定、CR0フラグ、32bitコードへジャンプ
その先へ ▶

本格的なOSを作るなら最終的にプロテクトモード（32bit）へ移行します。ここからBIOSは使えなくなり、自分でドライバを書く必要があります。

; プロテクトモード移行の骨格
    cli
    lgdt [gdt_descriptor]   ; GDTを登録
    mov eax, cr0
    or  eax, 0x1
    mov cr0, eax            ; PE bit ON
    jmp 0x08:protected_mode ; Far jump でCSを更新

[BITS 32]
protected_mode:
    ; ここから32bitコード
    ; BIOSは使えない、4GBアドレス空間が使える

参考: 「os-dev.org wiki」「Writing a Simple Operating System — From Scratch (Nick Blundell)」が定番の資料です。
